# Project Development Guidelines

This template follows a set of guidelines to ensure consistency, maintainability, and ease of collaboration.
Below are the key principles and practices that contributors should adhere to when developing from this template.


## 1. Project Philosophy & Stack

* **Package Manager**: `uv` (Universal Virtualenv) - A modern, fast, and user-friendly package manager for Python projects.
* **Python Version**: over `3.12`
* **Linting & Formatting**: `Ruff`
* **Testing**: `Pytest`

## 2. Directory Structure (Flat Layout)

This template adopts a 


```text
project-root/
├── docs  # This directory contains all documentation files for the project, organized in a flat structure for easy access and maintenance.
│   ├── __init__.py
│   ├── CONTRIBUTING.md
│   ├── gen_ref_pages.py
│   ├── index.md
│   └── reference.md
├── src
│   ├── configs  # This directory contains configuration files and settings for the project, such as environment variables, database configurations, and application settings.
│   │   ├── __init__.py
│   │   └── settings.py
│   ├── modules  # This directory contains the core modules and components of the application, organized by feature or functionality.
│   │   ├── first_module
│   │   │   ├── __init__.py
│   │   │   └── first_util.py
│   │   └── __init__.py
│   ├── __init__.py
│   └── main.py  # This is the main entry point of the application, where the core logic and execution flow are defined.
├── tests  # This directory contains all test cases for the project, organized by module or feature.
│   ├── __init__.py
│   └── test_default_utils.py
├── tools   # This directory contains utility scripts and tools that assist in development, testing, or deployment.
│   └── first_tool.py
├── .gitignore  # This file specifies intentionally untracked files to ignore, such as build artifacts, virtual environments, and sensitive information.
├── .python-version  # This file specifies the Python version to be used for the project, ensuring consistency across different development environments.
├── Makefile  # This file defines common tasks and commands for the project, such as building, testing, and cleaning.
├── mkdocs.yml  # This file configures the MkDocs documentation generator, specifying the site name, theme, and navigation structure.
├── pyproject.toml  # This file defines the project metadata, dependencies, and build configuration for the `uv` package manager.
├── README.md  # This file provides an overview of the project, installation instructions, usage examples, and contribution guidelines.
└── uv.lock   # This file is generated by `uv` and should be committed to version control.
```
